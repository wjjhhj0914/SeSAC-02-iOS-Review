# 02. UI Components Basic & Interface Builder

iOS 앱의 화면을 구성하는 기초 UI 객체들과 Xcode 인터페이스의 핵심 원리를 정리한다.

## 1. Xcode 인터페이스의 이해

* **Navigators(좌측 영역)**: 프로젝트의 파일 구조, 검색, 경고/에러 등을 확인하는 관제탑.
* **Storyboard(Interface Builder)**: 시각적으로 UI를 배치하는 작업 공간.
* **Document Outline(좌측 리스트)**: 현재 화면에 배치된 요소들의 계층 구조(Outline)를 보여줌.
  * **중요 (View 계층)**: 리스트에서 **아래에 있을수록 화면상에서는 가장 (앞)위**에 보여진다.
* **Inspector(우측 영역)**: 선택한 객체의 상세 설정을 변경함.
  * **Attributes Inspector**: 글자 색, 배경색, 상태 등 객체 고유의 속성을 설정.
  * **Size Inspector**: 객체의 크기, 위치, 레이아웃 제약 조건을 설정.

## 2. 앱의 시작과 구성

* **LaunchScreen(Splash Screen)**: 앱의 '대기실'. 앱이 메모리에 로드되는 동안 잠깐 보여주는 스플래시 화면임. (로직 작성 불가)
* **Main**: 사용자가 실제로 마주하게 되는 앱의 첫 화면이자 전체적인 구조가 시작되는 곳.
* **ViewController**: iOS 화면 구성의 핵심. **뷰컨트롤러라는 도화지 위에 여러 뷰를 얹어서 화면을 구성함.**

## 3. UIView: 태초의 시작점

모든 UI 요소(레이블, 버튼 등)의 공통 조상.

* **View 탭**: Attribute Inspector에서 어떤 객체를 선택하든 'View'라는 공통 설정 탭이 있다. 이는 모든 객체가 결국 UIView에서 시작되었음을 뜻한다.
* **계층 구조**: Superview(부모)와 Subview(자식) 관계로 이루어진다.
* **주요 속성**: 배경색(Background), 투명도(Alpha), User Interaction Enabled(클릭 가능 여부) 등.

## 4. 핵심 UI 컴포넌트 상세

### ① UILabel (텍스트 표시)

* 사용자가 읽기만 할 수 있는 텍스트 영역.
* `Lines`를 0으로 설정하면 줄 수 제한 없이 텍스트 길이에 따라 자동으로 늘어남.

### ② UIButton (클릭 이벤트)

* **클릭이 중요!** 사용자의 터치에 반응하며, `UIControl`을 상속받음.
* **UIButton 버전별 차이: iOS 15 미만 vs iOS 15 이상**

| 구분 | iOS 15 미만 (Legacy) | iOS 15 이상 (Modern) |
| :--- | :--- | :--- |
| **기본 스타일** | 주로 Plain (파란 글씨만 있는 형태) | **Filled, Tinted, Gray, Plain** 스타일 제공 |
| **배경 및 모서리** | 배경색 설정 후 Corner Radius 코드로 조절 | `Filled` 등 스타일 선택 시 즉시 적용 |
| **이미지/텍스트 간격** | `Edge Insets` 값을 수동 계산 (매우 번거로움) | `Padding` 속성으로 수치만 입력하여 간편 조절 |
| **이미지 배치** | 기본은 왼쪽 (우측 배치 시 커스텀 클래스 필요) | 상, 하, 좌, 우 위치를 Inspector에서 클릭으로 변경 |
| **서브타이틀 지원** | 기본 지원 안 함 (커스텀 레이블 추가 필요) | **Subtitle** 항목 기본 제공 (두 줄 버튼 가능) |
| **로딩 인디케이터** | 인디케이터 객체를 생성하여 직접 코드 배치 | `Shows Activity Indicator` 체크로 즉시 활성화 |

### ③ UIImageView (이미지 표시)

* **Content Mode**
  * `Aspect Fit`: 비율 유지하며 전체 보여주기.
  * `Aspect Fill`: 꽉 채우기 (비율 유지하되 잘릴 수 있음).

  ### ④ UITextField (입력창)

  * 한 줄만 입력을 받으며, 보안 입력(비밀번호)이나 키보드 타입 설정이 가능함.

## 5. Assets와 리소스 관리

### ① Assets

아이콘, 이미지, 컬러 등 앱에 필요한 모든 리소스를 관리하는 저장 공간.

* **AppIcon**: 앱의 얼굴인 아이콘을 설정하는 곳.
  * **주의**: 이 영역의 이름(`AppIcon`)을 임의로 수정하면 앱이 아이콘을 인식하지 못한다.
  * **경고 표시**: 규격(Size)에 맞지 않는 이미지를 넣으면 우측 하단에 노란색 경고가 뜬다. 정확한 픽셀 사이즈를 지키는 것이 중요하다.
  * **Appearance(모드 대응)**: 하나의 이미지나 컬러에 대해 세 가지 설정을 지원한다.
    * **Any Appearance**: 라이트 모드 및 다크 모드를 지원하지 않는 구형 OS용 기본값.
    * **Dark**: 다크 모드일 때 보여줄 리소스.
    * **Tinted**: (iOS 18+ 최신) 홈 화면 아이콘 편집 시 사용자가 설정한 강조색에 대응하는 리소스.

### ② 앱 아이콘의 진화

* **IconComposer & WWDC**: 애플은 WWDC를 통해 개발자들이 '리퀴드 글라스' 효과 등 애플 특유의 심미성을 유지하며 아이콘을 만들 수 있도록 디자인 규칙과 도구를 가이드라인으로 제공한다.
* **Alternative App Icons(다중 아이콘)**: 예전에는 앱 아이콘이 고정이었지만, 이제는 사용자의 상태나 이벤트에 따라 아이콘을 바꿀 수 있다.
  * _예: 듀오링고의 캐릭터가 공부를 안 하면 녹아내리는 모습으로 아이콘이 변하는 연출._

### ③ 이미지 해상도와 App Thinning

App Thinning은 [01_App_Environment.md](01_App_Environment.md)에서 정리를 했기 때문에 간략하게 언급하고 넘어간다.

* **1x, 2x, 3x의 존재 이유**: 아이폰마다 디스플레이의 픽셀 밀도(Density)가 다르기 때문이다.
* **App Thinning(애플리케이션 슬라이싱)**: 모든 해상도의 이미지를 다 설치하면 용량이 커지므로, 사용자 디바이스에 **딱 맞는 배수(예: iPhone 15 Pro는 3x)의 이미지 전용 패키지만 설치**되도록 **최적화**하는 기술이다. 이 때문에 기기별로 앱스토어 설치 용량이 다르게 나타난다.
  * **슬라이싱(Slicing)**: 기기별 최적화된 바이너리만 잘라내어 설치하는 기술.
  * **온디맨드 리소스(On-Demand Resources)**: 앱 설치 시 모든 용량을 받지 않고, 특정 레벨이나 단계가 되었을 때 추가 리소스를 서버에서 내려받는 방식. (최대 수 GB까지 지원)
  * **비트코드 방식**: 이거는 최근에 deprecated되어서 신경 안 써도 됨.

## 6. 애플의 디자인 철학과 크기 제약

**Intrinsic Content Size (본질적인 콘텐츠 크기)**
* 레이블이나 버튼처럼 내부 콘텐츠에 따라 스스로 결정하는 **본연의 크기**를 말한다.

**높이 설정의 비밀**
* `TextField`, `Segmented Control`, `Switch` 등은 높이 조절이 제한적이다.
* **이유**: 애플은 사용자가 어떤 앱을 써도 동일한 터치 경험과 표준 규격의 가독성을 느끼길 원하기 때문이다. _(이런 점들이 iOS 앱 개발을 하고 싶어지게 하는 점들..)_

---

아무튼 이 부분에서 가장 중요한 것.

* **iOS 화면 구성**: 뷰컨트롤러(VC) → 뷰(View) 얹기 → 컴포넌트 배치.
* **클릭 인터렉션**: 사용자가 누를 수 있는 요소는 `Control` 속성을 가진다.
* **버전 대응**: iOS 15 이상의 프로젝트라면 가급적 최신 `Configuration` 기반 버튼을 사용하는 것이 생산성에 유리하다.
